render (line 79)

render the actual view and layout. normally done at the end of a action of a controller

  1.  1. all helpers are constructed here, very late, so we dont accidently waste cpu cycles
  2.  2. all variables, helpers and params given from the controller are extracted to global namespace
  3.  3. the actual view-template is rendered
  4.  4. the content of the rendered view is rendered into the layout

  • return: html of the view
  • access: public
